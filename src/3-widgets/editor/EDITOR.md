## https://www.blocknotejs.org/docs/editor-basics/setup

`useCreateBlockNote` 훅 옵션 상세 설명

* **`animations`**:
    * **역할:** 블록의 변경 사항(들여쓰기, 목록 생성, 제목 변경 등)에 애니메이션 효과를 적용할지 여부를 결정합니다.
    * **기본값:** `true`
    * **설명:** 사용자 경험을 부드럽게 만들기 위해 블록이 이동하거나 스타일이 변경될 때 시각적인 전환 효과를 제공합니다. 애니메이션을 원치 않으면 `false`로 설정할 수 있습니다.

* **`collaboration`**:
    * **역할:** 실시간 협업 기능을 활성화하고 설정하는 옵션입니다.
    * **참고:** "Real-time Collaboration" 문서를 참조하여 자세한 설정 방법을 확인해야 합니다.
    * **설명:** 여러 사용자가 동시에 동일한 문서를 편집할 수 있도록 하는 Yjs와 같은 기술을 통합할 때 사용됩니다.

* **`comments`**:
    * **역할:** 댓글 기능을 설정합니다. 이를 사용하려면 `threadStore`가 필요합니다.
    * **참고:** "Comments" 문서를 참조하여 자세한 설정 방법을 확인해야 합니다.
    * **설명:** 특정 블록이나 텍스트 범위에 대한 사용자 간의 댓글 기능을 추가할 때 사용됩니다.

* **`defaultStyles`**:
    * **역할:** BlockNote 내부에서 사용되는 `<p>`, `<li>`, `<h1>` 등의 HTML 요소에 기본 폰트와 스타일 초기화를 적용할지 여부를 결정합니다.
    * **기본값:** `undefined`일 경우 `true`
    * **설명:** BlockNote는 자체적으로 몇 가지 기본 스타일을 제공하여 에디터의 일관된 모양을 유지합니다. 만약 여러분의 애플리케이션에 고유한 전역 스타일이 있고 BlockNote의 기본 스타일과 충돌하는 것을 원치 않는다면 `false`로 설정할 수 있습니다.

* **`dictionary`**:
    * **역할:** 에디터의 현지화(Localization / i18n)를 위한 문자열을 제공합니다.
    * **참고:** "Localization / i18n example" 및 "Custom Placeholders" 문서를 참조하세요.
    * **설명:** 에디터 UI에 표시되는 텍스트(예: 메뉴 항목, 플레이스홀더 텍스트)를 다양한 언어로 번역하거나 커스터마이징할 때 사용됩니다.

* **`disableExtensions` (고급)**:
    * **역할:** BlockNote에서 기본적으로 사용되는 TipTap 확장 기능을 이름에 따라 비활성화합니다.
    * **설명:** TipTap은 BlockNote의 기반이 되는 텍스트 에디터 프레임워크입니다. 특정 내장 기능을 비활성화해야 할 때 사용되는 고급 옵션입니다. 예를 들어, 특정 포매팅 옵션을 제공하고 싶지 않을 때 유용할 수 있습니다.

* **`domAttributes`**:
    * **역할:** 에디터 내의 다양한 DOM 요소에 추가할 HTML 속성을 포함하는 객체입니다.
    * **참고:** "Adding DOM Attributes" 문서를 참조하세요.
    * **설명:** 에디터의 특정 부분(예: 최상위 에디터 컨테이너, 개별 블록)에 `class`, `data-*` 속성 등을 추가하여 사용자 정의 스타일링이나 JavaScript 동작을 적용할 때 사용됩니다.

* **`dropCursor`**:
    * **역할:** 블록을 드래그 앤 드롭할 때 사용되는 대체 지시자(인디케이터)입니다. `ProseMirror drop cursor` 또는 `Column Blocks`가 활성화되었을 때 수정된 버전을 사용합니다.
    * **설명:** 사용자가 블록을 끌어서 놓을 때, 블록이 어느 위치에 삽입될지 시각적으로 보여주는 커서의 모양과 동작을 제어합니다.

* **`initialContent`**:
    * **역할:** 에디터가 처음 생성될 때 포함될 콘텐츠를 `PartialBlock` 배열로 나타냅니다.
    * **설명:** 에디터를 특정 내용으로 미리 채우고 싶을 때 사용합니다. 예를 들어, 데이터베이스에서 불러온 기존 문서 내용을 에디터에 로드할 때 이 옵션을 사용합니다.

* **`pasteHandler`**:
    * **역할:** 기본 붙여넣기 동작을 재정의하는 데 사용할 수 있는 함수입니다.
    * **참고:** "Paste Handling" 문서를 참조하세요.
    * **설명:** 외부 콘텐츠(예: 웹 페이지, 워드 문서)를 에디터에 붙여넣을 때, BlockNote의 기본 붙여넣기 처리 방식이 아닌 사용자 정의 로직을 적용하고 싶을 때 사용됩니다. 예를 들어, 특정 HTML 태그를 제거하거나 특정 방식으로 변환할 수 있습니다.

* **`resolveFileUrl`**:
    * **역할:** 파일 URL을 표시/다운로드를 위해 해결하는 함수입니다. 인증된 URL을 생성하거나 사용자 정의 프로토콜을 구현하는 데 유용합니다.
    * **설명:** 에디터 내에서 참조되는 이미지나 다른 파일의 URL이 특정 로직(예: 인증 토큰 추가, CDN 경로 변환)을 거쳐야 할 때 사용됩니다.

* **`resolveUsers`**:
    * **역할:** 댓글 기능을 위해 사용자 정보를 해결하는 함수입니다.
    * **참고:** "Comments" 문서를 참조하세요.
    * **설명:** 댓글에서 사용자 멘션(@user)을 구현하거나 사용자 프로필 정보를 표시할 때, 사용자 ID를 실제 사용자 이름이나 아바타와 같은 정보로 변환하는 데 사용됩니다.

* **`schema`**:
    * **역할:** 사용자 정의 블록, 스타일 또는 인라인 콘텐츠로 에디터를 확장하려는 경우 사용되는 에디터 스키마입니다.
    * **참고:** "Custom Schemas" 문서를 참조하세요.
    * **설명:** BlockNote가 기본적으로 제공하지 않는 새로운 종류의 블록(예: 카드 블록, 임베드 블록)을 만들거나, 기존 블록에 새로운 스타일(예: 특정 색상 강조)을 추가할 때 사용되는 고급 기능입니다.

* **`setIdAttribute`**:
    * **역할:** 블록에 `data-id` 속성뿐만 아니라 HTML `id` 속성도 렌더링할지 여부를 결정합니다.
    * **기본값:** `false`
    * **설명:** 이전에 설명했듯이, 특정 블록으로의 앵커 링크나 `document.getElementById`와 같은 표준 DOM API를 사용하여 블록을 조작해야 할 때 `true`로 설정합니다.

* **`sideMenuDetection`**:
    * **역할:** 마우스 커서 위치가 "블록 사이드 메뉴" 및 블록 드래그 앤 드롭을 표시하기 위해 에디터 경계 상자에 고정될지 여부를 결정합니다.
        * `viewport`로 설정하면, 커서가 뷰포트 내 어디에 있든 가장 가까운 블록 옆에 사이드 메뉴가 표시됩니다. 블록 드롭도 에디터 경계 상자에 고정됩니다.
        * 그렇지 않으면, 사이드 메뉴는 커서가 에디터 경계 내에 있을 때만 표시되며, 블록은 에디터를 호버링할 때만 드롭될 수 있습니다.
        * 여러 에디터를 사용하려면 반드시 `editor`로 설정해야 합니다.
    * **기본값:** `viewport`
    * **설명:** 에디터 옆에 나타나는 + 버튼 메뉴(블록 사이드 메뉴)와 블록 드래그 동작이 언제 활성화될지 정의합니다. 특히 한 페이지에 여러 BlockNote 에디터를 사용할 때 중요한 설정입니다.

* **`tabBehavior`**:
    * **역할:** 탭 키를 누를 때 키보드 접근성을 위해 도구 모음을 탐색할지 여부를 결정합니다.
        * `"prefer-navigate-ui"`로 설정하면, 사용자가 Tab 키를 사용하여 도구 모음을 탐색할 수 있습니다. Escape를 누르면 에디터로 다시 포커스되고, 이후 Tab은 블록을 들여쓰기합니다.
        * `"prefer-indent"`는 Tab이 항상 블록을 들여쓰게 합니다.
    * **기본값:** `"prefer-navigate-ui"`
    * **설명:** 키보드만을 사용하여 에디터를 조작하는 사용자들을 위한 접근성 설정입니다. Tab 키가 포커스를 이동시킬지, 아니면 텍스트 들여쓰기 기능을 수행할지 결정합니다.

* **`tables`**:
    * **역할:** 테이블 기능에 대한 설정입니다. `splitCells` (셀 분할), `cellBackgroundColor` (셀 배경색), `cellTextColor` (셀 텍스트 색상), `headers` (헤더)와 같은 고급 테이블 기능을 활성화할 수 있습니다.
    * **설명:** 에디터 내에서 테이블을 사용할 때, 더 풍부한 편집 기능을 제공할지 여부를 설정합니다.

* **`trailingBlock`**:
    * **역할:** 사용자가 블록을 입력하거나 편집할 때 다음 줄에 자동으로 새 블록을 생성하는 기능을 비활성화하려면 `false` 값을 전달합니다.
    * **기본값:** `undefined`일 경우 `true`
    * **설명:** 일반적으로 BlockNote는 마지막 블록 뒤에 항상 빈 단락 블록을 자동으로 유지하여 사용자가 쉽게 계속 입력할 수 있도록 합니다. 이 동작을 원치 않을 때 비활성화할 수 있습니다.

* **`uploadFile`**:
    * **역할:** 파일 업로드를 처리하고 최종적으로 업로드된 파일의 URL을 반환하는 함수입니다. 주로 이미지 블록에 사용됩니다.
    * **설명:** 사용자가 에디터에 이미지를 삽입할 때, 실제 파일을 서버에 업로드하고 그 URL을 받아와 에디터에 표시하기 위한 커스텀 로직을 구현할 때 사용합니다.

#### deps (의존성 배열)

* **역할:** 내부적으로 `useMemo`에 전달되는 의존성 배열입니다. 이 배열이 변경될 때만 새로운 에디터 인스턴스가 생성됩니다.
* **설명:** React 훅의 `deps` 배열과 동일한 개념입니다. 에디터 설정(`options`)이나 다른 외부 변수가 변경될 때 에디터를 다시 초기화해야 하는 경우에 사용합니다. 불필요한 에디터 재생성을 방지하여 성능을 최적화하는 데 도움이 됩니다.

`<BlockNoteView>` 컴포넌트 Props 상세 설명

* **`editor`**:
    * **역할:** 렌더링할 `BlockNoteEditor` 인스턴스. 이 prop은 **필수**입니다.
    * **설명:** `useCreateBlockNote()` 훅을 통해 얻은 에디터 객체를 이곳에 전달해야 `<BlockNoteView>`가 어떤 에디터를 렌더링할지 알 수 있습니다.

* **`editable`**:
    * **역할:** 에디터가 편집 가능한 상태인지 여부를 결정합니다.
    * **설명:** `true`로 설정하면 사용자가 콘텐츠를 수정할 수 있고, `false`로 설정하면 읽기 전용 모드가 되어 콘텐츠를 볼 수만 있습니다. 미리보기 모드나 콘텐츠를 잠궈야 할 때 유용합니다.

* **`onSelectionChange`**:
    * **역할:** 에디터의 선택 영역(텍스트 선택, 블록 선택 등)이 변경될 때 호출되는 콜백 함수입니다.
    * **설명:** 예를 들어, 사용자가 특정 텍스트를 선택했을 때 선택된 텍스트에 대한 정보를 얻거나, 선택 영역에 따라 특정 UI를 업데이트해야 할 때 사용합니다.

* **`onChange`**:
    * **역할:** 에디터의 내용(문서)이 변경될 때 호출되는 콜백 함수입니다.
    * **설명:** 에디터의 콘텐츠가 수정될 때마다 트리거됩니다. 변경된 내용을 데이터베이스에 저장하거나, 실시간으로 다른 사용자에게 동기화하는 등의 로직을 구현할 때 핵심적으로 사용됩니다.

* **`theme`**:
    * **역할:** 에디터의 테마(라이트 또는 다크 모드 등)를 설정합니다.
    * **참고:** "Themes" 문서를 참조하여 자세한 설정 방법을 확인해야 합니다.
    * **설명:** `'light'`, `'dark'` 문자열로 설정하거나, 직접 정의한 `Theme` 객체를 전달하여 에디터의 색상, 폰트 등을 커스터마이징할 수 있습니다.

* **`formattingToolbar`**:
    * **역할:** 텍스트 선택 시 나타나는 **포매팅 툴바**를 활성화할지 여부를 결정합니다.
    * **설명:** 볼드, 이탤릭, 밑줄, 링크 등 텍스트 서식을 지정할 때 사용되는 작은 팝업 툴바를 표시할지 여부를 제어합니다.

* **`linkToolbar`**:
    * **역할:** 링크 클릭 시 나타나는 **링크 툴바**를 활성화할지 여부를 결정합니다.
    * **설명:** 링크를 클릭했을 때 링크 편집, 열기, 제거 등의 옵션을 제공하는 툴바를 표시할지 여부를 제어합니다.

* **`sideMenu`**:
    * **역할:** 블록 옆에 나타나는 **블록 사이드 메뉴**를 활성화할지 여부를 결정합니다.
    * **설명:** 마우스를 블록 옆에 가져가면 나타나는 `+` 버튼과 드래그 핸들을 통해 블록을 추가, 삭제, 이동하는 기능을 제공하는 메뉴를 표시할지 여부를 제어합니다.

* **`slashMenu`**:
    * **역할:** 슬래시(`/`) 입력 시 나타나는 **슬래시 메뉴**를 활성화할지 여부를 결정합니다.
    * **설명:** `/`를 입력하여 블록 타입을 빠르게 변경하거나, 이미지, 테이블 등을 삽입할 수 있는 명령 팔레트와 같은 메뉴를 표시할지 여부를 제어합니다.

* **`emojiPicker`**:
    * **역할:** 이모지 선택기 기능을 활성화할지 여부를 결정합니다.
    * **설명:** 사용자가 이모지를 쉽게 삽입할 수 있도록 도와주는 UI를 표시할지 여부를 제어합니다.

* **`filePanel`**:
    * **역할:** 파일 툴바를 활성화할지 여부를 결정합니다.
    * **설명:** 주로 이미지나 다른 파일을 삽입할 때 사용되는 파일 관련 UI 요소를 표시할지 여부를 제어합니다.

* **`tableHandles`**:
    * **역할:** 테이블 편집 시 나타나는 **테이블 핸들**을 활성화할지 여부를 결정합니다.
    * **설명:** 테이블의 행/열을 추가, 삭제하거나 테이블 전체를 선택하는 데 사용되는 UI 요소(핸들)를 표시할지 여부를 제어합니다.

* **`comments`**:
    * **역할:** 기본 댓글 UI 기능을 활성화할지 여부를 결정합니다.
    * **설명:** `useCreateBlockNote`의 `comments` 옵션과 연동하여 에디터 내에서 댓글을 표시하고 상호작용하는 UI를 제공할지 여부를 제어합니다.

* **`children`**:
    * **역할:** `<BlockNoteView>`에 자식 요소를 전달하여 툴바, 메뉴 또는 기타 UI 구성 요소를 생성하거나 커스터마이징할 수 있습니다.
    * **참고:** "UI Components" 문서를 참조하여 자세한 방법을 확인해야 합니다.
    * **설명:** BlockNote가 제공하는 기본 UI 외에 사용자 정의 UI 요소를 에디터 내부에 추가하거나, 기본 UI를 특정 방식으로 확장하고 싶을 때 사용됩니다.

* **HTMLAttributes`<HTMLDivElement>`**:
    * **역할:** `<BlockNoteView>`에 전달되는 추가적인 props는 BlockNote가 내부적으로 렌더링하는 최상위 HTML `div` 요소로 전달됩니다.
    * **설명:** 예를 들어 `<BlockNoteView className="my-custom-editor" style={{ border: '1px solid gray' }} />`와 같이 `className`이나 `style`과 같은 표준 HTML 속성을 직접 전달하여 에디터 컨테이너의 스타일을 지정할 수 있습니다.

### Uncontrolled Component (비제어 컴포넌트)

BlockNoteView 컴포넌트는 **비제어 컴포넌트(uncontrolled component)**입니다. 이는 중요한 개념입니다.
* **의미:** React에서 비제어 컴포넌트는 **자체적으로 내부 상태를 관리**하며, 부모 컴포넌트로부터 props로 직접 상태를 전달받지 않습니다. `<input type="file">`이나 일반적인 `<textarea>`의 `defaultValue` prop과 유사합니다.
* **에디터 콘텐츠 관리:** 에디터의 콘텐츠(내부 상태)를 props로 직접 전달하지 않습니다. 대신, `useCreateBlockNote` 훅의 `initialContent` 옵션을 사용하여 **초기 콘텐츠만 설정**합니다.
* **상태 관리의 복잡성:** BlockNote는 에디터 상태를 관리하고 성능을 최적화하는 복잡한 작업을 내부적으로 처리합니다. 개발자는 이 복잡성을 직접 다룰 필요 없이, 에디터 API를 사용하여 콘텐츠를 읽거나 조작할 수 있습니다.
* **콘텐츠 접근:** 에디터 콘텐츠가 변경될 때 `onChange` 콜백을 통해 알림을 받고, 필요할 경우 `editor.getBlocks()`와 같은 **에디터 API**를 사용하여 현재 에디터의 콘텐츠에 접근할 수 있습니다.
